> We don't want a QuestionBankEntity. question bank is a term we are using abstractly, not one that should appear in the domain. what we want is an abstract class AssessmentQuestion, which has a String category (This will be used to filter question subjects/categories), and a String question.
> 
> Then there will be 4 concrete classes from this abstract class. MCQuestion,TextQuestion, DocQuestion, and Group Question.
> 
> MCQuestion will extned the AssessmentQuestion by including a List<String> options, and List<String> correctAnswers.

> DocQuestion will extend AssessmentQuestion without adding much (at this stage)
> GroupQuestion will extend TextQuestion and have an ordered List(or other suitable collection)<TextQuestion> followUpQuestions, which will essentially ask a text question and ask follow up questions.
