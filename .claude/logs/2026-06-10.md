**Prompt:**
> We were planning on starting phase 6-8, and in fact started the scaffolding for phase 6 already, but we were called into a meeting to prioritize some other changes first, before starting phase 6. So we need some new specs, and an adjustment in current specs.
> 
> What we need: 
> 
> QuestionBanks(QB): Instead of each AssessmentQuestion(AQ) having a String category, we want an AQ to be linked to possibly many QB's. The QB's would have a name to give identity to the QB. E.g. "Java: Basics", "Java: Strings", "OOP", etc. One QB can have many AQ's, and one AQ can belong to many QB's.
> 
> McqPlusQuestion: We would like to have an extension of an McqQuestion, where it would follow the same rules, except that there would be one follow up TextQuestion. This is too see if the correct answer was just by chance, or if they actually understood the work.
> 
> Marking: Questions should have attached marks on them when generated/created. These would extend to DocQuestion, and TextQuestion. For a group question, the mark would just be the total of the "child" questions. MCQ's would always just be worth one mark, however where an McqPlus is used, there will be one mark for the mcq aspect, and then a mark attached to the text question.
> 
> Separation: I've noticed that on a group question, that there is a list of TextQuestions that the user can select to be the "child" questions for that group question. This is not what i want. Just like adding options to an MCQ question, i want to input follow-up questions each time, not select from all available text questions. The same must be true for the McqPlusQuestion. Additionally when i am searching or filtering through TextQuestions, I do not want to see any that are "child" questions of a group or mcq question. These must be completely stand-alone questions
> 
> Please use AskUserQuestions for any clarity of understanding, and feel to use it as many times as needed. I want to get the specs right before starting.
